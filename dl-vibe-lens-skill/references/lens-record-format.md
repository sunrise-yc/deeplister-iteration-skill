# Vibe Lens 记录格式

当 `docs/iteration-record.md` 缺失、结构损坏，或需要新增字段时，读取这份参考。第一次使用请运行脚本的 `--init`，不要让用户手动从零创建文件。

## 默认路径

```text
docs/iteration-record.md
```

脚本也会读取旧路径：

```text
docs/迭代记录.md
```

这是为了兼容更早的 DeepLister 和 `vibe-iteration` 记录。

## 记录语言

新记录应该默认跟随当前对话语言。

- 中文对话：使用 `--record-language zh`，或省略参数让 CLI 默认生成中文模板。
- 英文对话：使用 `--record-language en`。
- `record_language: "auto"` 表示 agent 写入新记录时跟随当前对话语言。
- HTML 报告不能偷偷把源记录里的英文问题机器翻译后展示。
- 如果已有英文条目需要中文化，agent 必须明确执行“翻译并写回记录”或“生成翻译缓存”。

## 必须保留的章节

脚本按二级标题读取数据。不要改名这些标题，也不要把 `## 问题池` 里的表格搬到别处。

```md
# Vibe Lens 记录

## 保护说明
## 当前产品方向
## 问题池
## 当前工作
## 追问流程专项记录
## 迭代记录
```

旧英文标题仍可兼容：

```md
## Issue Pool
## Active Work
## Follow-up Flow Notes
## Iteration Log
```

## 问题池

在 `## 问题池` 下放一个 Markdown 表格。

推荐列：

```md
| 编号 | 问题 | 来源 | 状态 | 证据 | 关联文件 | 验证 | 解释状态 |
|---|---|---|---|---|---|---|---|
| VL-001 | 简短问题标题 | operator | open | 这个问题为什么存在 | src/app.py | python -m unittest | 已解释 |
```

“问题有证据”不是说问题一定正确，而是说它可追溯、可核对、可复盘。证据可以来自用户反馈、Git diff、测试结果、截图、日志、历史决策或运行现象。

兼容旧列：

```md
| ID | Issue | Impact | Priority | Status | Next Step |
| 编号 | 问题 | 影响 | 优先级 | 状态 | 下一步 |
```

`Priority` 只保留为旧字段展示信息。Vibe Lens 可以显示它，但不能根据它给任务排序。

常用状态：

| 状态 | 含义 |
|---|---|
| `open` | 问题仍需要被看见 |
| `discussing` | 正在澄清 |
| `in progress` | 已开始相关工作 |
| `blocked` | 需要用户输入或外部条件 |
| `resolved` | 已处理并验证 |
| `done` | 旧完成状态 |

中文状态如 `待处理`、`进行中`、`阻塞`、`已解决`、`完成` 也可以。

## 当前工作

`## 当前工作` 用来展示并行 AI coding 对话可能碰到的文件或区域。

```md
| 会话 | 任务 | 涉及文件 | 状态 | 备注 |
|---|---|---|---|---|
| 2026-07-03 | 调整报告模板 | dl-vibe-lens-skill/assets/report_template.html | in progress | 当前对话负责这块 |
```

这一节只生成冲突线索，不告诉操作者哪个任务必须先做。

## 迭代记录

在 `## 迭代记录` 下追加新条目，最新条目放在靠前位置。

```md
### YYYY-MM-DD: 迭代标题
目标：
-

提出的问题：
-

证据：
-

代码变化：
-

验证：
-

未完成：
-
```

避免无标签的建议语言。如果必须给建议，要标成“Agent 判断”或“操作者决策”，不能伪装成 Vibe Lens 事实。

## 追问流程专项记录

`## 追问流程专项记录` 用来记录行为案例、追问模式、证据缺口和冲突线索。

```md
| 场景 | 当前表现 | 问题 | 优化方向 |
|---|---|---|---|
| 两个 AI 对话同时改一个项目 | 每个对话上下文不同 | 容易冲突 | 大范围修改前展示活跃文件或区域 |
```

## 什么时候生成 HTML 报告

出现这些情况时，生成 HTML：

- 问题池已经不好扫读。
- 代码变化需要可视化 diff 统计。
- 迭代方向发生多次变化。
- 用户要求 review、沙盘、dashboard 或可视化总览。
- 多个 AI 会话可能并行修改同一项目。
